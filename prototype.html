Two screens, one demo

Login earns the moment — premium, calm, BYO-keys honest in the footnote. Home delivers the wedge — agent recommendations and critic-pairing are the first thing your eye lands on, not buried in metadata.

What the home screen surfaces

Recommendation on every PR The wedge made visible: agent commits to a verdict + confidence + one thing to read. Tap to open the full review form factor.
Critic-pair on plans Planner → critic relationship rendered as the agent dots. Unresolved critique notes show as a count — disagreement is the signal.
Trust % on agent badges Per-agent accept-rate from your own history. The data moat that takes a year to build, made visible in 11px.
Loupe-inferred suggestions Issues carry a spark-marked hint (“dispatch now” vs “plan first”) drawn from prior outcomes. Not static copy — product intelligence.

Why this aesthetic

Cream-on-deep-navy with the indigo glow signals refinement, not operations. Loupe is for engineers stepping into a director's role — the surface should feel like a calm command center, not a Splunk dashboard. The Caveat wordmark and Familjen Grotesk pairing pulls warmth out of the indigo to keep the product approachable.

Interaction variants to test

The phone mock stays stable; the rationale below changes so the design stays critique-friendly.

Split rail + inbox
  • Best balance: one homescreen, two mental buckets.
  • Running tasks stay visible without polluting the decision queue.
  • Recommended default for v1.
Pure single stream
  • Cleaner conceptually, but live tasks and decisions fight for rank.
  • Works only if “currently running” is never more than one or two items.
  • More Linear-like, less operations-like.
Queue-first
  • Great for power users running many agents.
  • Risks making Loupe feel like a job runner instead of a decision surface.
  • Better as a second tab called Runs.

Open design questions

Should plan approvals live in the active rail or the attention inbox? My bias: both. Show the plan card in the active rail while it is fresh, but also rank it in the inbox if it blocks implementation. Should agent trust % default visible or live behind a tap on the agent badge? Probably visible — but it must never exceed two characters.
